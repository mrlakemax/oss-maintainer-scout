# Maintainer Support Application Guide

This project can help maintainers prepare evidence for programs that support open-source work, including Codex-style maintainer workflows.

## What oss-maintainer-scout Can Provide

- A repeatable report of repository readiness.
- A list of missing maintainer workflow files.
- Evidence that tests, CI, security policy, and contribution guidance exist.
- A roadmap for future issue, pull request, release, and security automation.

## What It Cannot Provide

- It cannot guarantee acceptance into any external program.
- It cannot create real adoption, stars, forks, or contributor history.
- It cannot truthfully turn a private or unused repository into a widely used public project.

## Honest Application Narrative

Use real facts from the repository:

- what the project does,
- who maintains it,
- public repository URL,
- license,
- release history,
- tests and CI,
- maintainer workflows that Codex or API credits would improve.

Good uses of API credits:

- issue triage,
- pull request review assistance,
- regression test generation,
- documentation maintenance,
- security review summaries,
- release note drafting.

Avoid claiming usage, ecosystem importance, or maintainer status that cannot be verified.
